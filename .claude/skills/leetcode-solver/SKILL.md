# LeetCode Solver Skill

## 功能概述

本skill帮助用户分析和解决LeetCode题目：
1. 识别用户输入的是题目号还是代码
2. 如果是题目号，访问LeetCode获取题目详情和最优Java解
3. 搜索相关学习资源（文章、视频、讨论区）
4. 整理并输出完整的学习资料

## 输入识别

### 判断输入类型

- **题目号**: 纯数字或"LC"开头的字符串，如 `1`, `206`, `LC 1`, `lc-206`
- **代码**: 包含Java代码片段的输入，如包含 `class Solution`, `public`, `void`, `int[]` 等

### 题目号格式标准化

将以下格式统一转换为LeetCode URL格式：
- `1` → `https://leetcode.cn/problems/two-sum/`
- `206` → `https://leetcode.cn/problems/reverse-linked-list/`
- `LC 1` 或 `lc 1` → `https://leetcode.cn/problems/two-sum/`

## 获取题目信息

### 重要说明：必须使用 LeetCode CN 版本

**禁止使用 leetcode.com（英文版）**，因为：
1. 英文版需要登录才能查看完整内容
2. 访问受限，经常返回403错误

**必须使用 leetcode.cn（中文版）**：
- 无需登录即可查看完整题目和题解
- 内容更稳定，访问成功率更高
- 支持直接获取Java代码

### 步骤1: 访问LeetCode题目页面

使用 chrome-devtools MCP 访问题目页面：

```
URL: https://leetcode.cn/problems/[题目slug]/
```

题目slug转换规则：
- 如果已知题号，使用题号查询获取slug
- 常见题目slug:
  - 1: two-sum
  - 2: add-two-numbers
  - 3: longest-substring-without-repeating-characters
  - 206: reverse-linked-list
  - 101: symmetric-tree
  - 104: maximum-depth-of-binary-tree

### 步骤2: 提取题目信息

从页面中提取：
1. **题目名称** (Title)
2. **题目编号** (Problem number)
3. **难度** (Difficulty: Easy/Medium/Hard)
4. **考察标签** (Topics/Tags)
5. **题目描述** (Description)
6. **示例** (Examples)
7. **约束条件** (Constraints)

### 步骤3: 获取Java最优解

在题目页面上找到：
1. **Solution** 标签页
2. 选择 **Java** 语言
3. 找到 **Top Solution** 或 **Optimal** 解法
4. 提取完整代码

如果没有找到最优解，可以尝试：
- 访问 Discuss 页面按votes排序
- 搜索 "Java solution" 或 "Java optimal"

## 搜索相关资源

### 搜索配置

资源搜索使用配置文件：`/Users/qinsx/project/AI/claude-skill/.claude/skills/leetcode-solver/search-config.json`

### 高质量来源（按优先级排序）

**强烈推荐**的资源来源：

| 优先级 | 来源 | 特点 |
|--------|------|------|
| 1 | **代码随想录** (programmercarl.com) | B站最火的算法教程，讲解详细，包含多种解法对比 |
| 2 | **灵茶山艾府** (bilibili/endlesscheng) | 算法竞赛选手视角，解法简洁有力，常有独到见解 |
| 3 | **LeetCode 官方** (leetcode.cn) | 官方提供的最优解，经过验证 |

**默认启用的来源**：代码随想录、灵茶山艾府、官方题解

### 使用方法

**方式一：修改配置文件**
编辑 `search-config.json` 中的 `enabled_types` 数组：
```json
// 推荐配置
"enabled_types": ["代码随想录", "灵茶山艾府", "官方题解"]

// 最小配置
"enabled_types": ["官方题解"]

// 全部来源
"enabled_types": ["代码随想录", "灵茶山艾府", "官方题解", "文章", "视频", "讨论区"]
```

**方式二：在输入中指定**
使用时可以在题目号后添加来源限制，例如：
- `15 三数之和 代码随想录` - 只看代码随想录
- `15 三数之和 灵茶山艾府` - 只看灵茶山艾府
- `15 三数之和 官方题解` - 只看官方题解

### 执行搜索

根据配置执行搜索，使用 WebSearch 或 WebFetch 工具获取资源。

### 资源验证（重要）

**每个搜索到的资源链接必须通过浏览器验证！**

1. **使用 chrome-devtools MCP 验证每个资源链接：**
   ```
   mcp__chrome-devtools__new_page: 打开链接
   mcp__chrome-devtools__take_snapshot: 获取页面内容
   ```

2. **验证标准：**
   - 页面能成功加载（状态码 200）
   - 页面有实质内容（非404/403/空页面）
   - 内容与题目相关

3. **验证流程：**
   ```
   for 每个搜索结果:
       1. 用 new_page 打开链接
       2. 等待页面加载 (wait_for)
       3. 用 take_snapshot 获取页面内容
       4. 检查页面是否有效
       5. 如果无效，标记为"❌ 无法访问"
       6. 关闭页面 (close_page)
   ```

4. **标记结果：**
   - ✅ 验证通过 - 保留该资源
   - ❌ 验证失败 - 从报告中移除该资源

5. **注意事项：**
   - 如果 Chrome 未运行，先启动 Chrome
   - 验证失败的资源不要写入报告
   - 确保每个写入报告的链接都是可访问的

### 资源筛选标准

优先选择：
- 近期发布的内容
- 高点赞/高浏览量
- 详细讲解
- 代码完整可运行

## 工作流程

### 完整执行流程

1. **识别输入**
   - 判断是题目号还是代码
   - 如果是代码，尝试识别题目（可能需要分析代码逻辑）

2. **获取题目信息**
   - 构建LeetCode URL
   - 访问页面提取信息
   - 获取Java最优解代码

3. **搜索相关资源**
   - 读取搜索配置 `search-config.json`
   - 检查用户输入是否有来源限制（如"只看官方题解"）
   - 根据配置的 enabled_types 执行搜索
   - 整理搜索结果

4. **验证资源链接**（重要）
   - 使用 chrome-devtools MCP 打开每个资源链接
   - 验证页面是否能正常加载并有内容
   - 标记验证结果（✅通过 / ❌失败）
   - 移除验证失败的链接

5. **生成报告**
   - 按输出格式整理所有信息（只包含验证通过的链接）
   - 保存到 `/Users/qinsx/project/AI/claude-skill/leetcode_target/{题号}-{题目slug}.md`
   - 输出给用户

6. **代码可视化（重要！自动执行）**
   - 获取 Java 最优解代码后，**必须**调用 `gen-solution-viz` skill 生成交互式可视化页面
   - 使用 Skill 工具调用：
     ```
     skill: "gen-solution-viz"
     args: "题目 15 三数之和, Java, 数组 双指针 排序, 代码: {Java代码}"
     ```
   - 生成完成后告知用户可视化页面的路径

## 注意事项

- 如果页面加载失败，尝试使用不同的方法获取信息
- 确保代码是最新的Java版本
- 资源链接需要验证有效性
- 如果用户输入的是代码而非题号，尝试分析代码推断题目
